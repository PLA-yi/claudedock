## Overview

把项目自己的端到端测试体系一次性长出来。沿着 38 条 v1 需求（E2E-01..05 / MVS-01..10 / LEAK-01..08 / KILL-01..04 / QUAL-01..08 + 51-09 / OBS-01..03）在 `tests/e2e/` 目录下接入 testcontainers-go + testify/suite + Scenario 声明式拓扑 + `waitFor` 条件等待 + ArtifactDumper 失败归档；CI 走 hosted ubuntu-24.04 双层 workflow（lint + e2e job + paths 守护 + 失败 PR 评论）；MVS / LEAK / KILL 三套真实跑路径的 e2e 用例骨架就位；Phase 51 把 9 项代码层质量加固一次性落地；Phase 52 把失败诊断做成一键工程。

里程碑通过三段式落地：

- **主线 A · 测试基础设施（Phase 45 + 52）** —— testcontainers-go + testify/suite + Scenario builder API + waitFor 4 变体 + ArtifactDumper 5 子目录 + `collect-artifacts.sh` 一键采集脚本 + CI 双层 workflow（hosted ubuntu-24.04 + paths 守护 + PR 评论）+ `scripts/lint-no-bare-sleep.sh` 双层守护。
- **主线 B · MVS / LEAK / KILL 三套 e2e 用例（Phase 46 / 47 / 48 / 49 / 50）** —— 黄金路径（bootstrap → 出口 IP / DNS / 默认拒绝 / CLI 错误码）+ 治理（到期自动停止 / 双绑互斥 / host-agent 心跳与恢复）+ kill-switch 核心（sing-box 崩溃 / resolv.conf 篡改）+ 8 条防泄漏不变量（DNS/DoT/ICMP/IPv6/IMDS/raw socket/link-local/capability）+ 4 条 kill-switch 压力（SIGKILL/tun0 down/Pumba netem/network disconnect）。
- **主线 C · 代码层质量加固（Phase 51）** —— verify.go 多源轮询 / 多目标泄漏检测 / 全 nameserver 校验 + namespace.go 探测窗口 functional option + worker_firewall_linux 全规则 counter + 显式 link-local drop + worker cap-drop NET_RAW（NET_ADMIN 折中保留） + `-race -shuffle=on -count=1` 默认化 + goleak.VerifyTestMain 接入 + 双绑互斥后端 pre-check（51-09 闭 Phase 47 D-47-3 gap）。
