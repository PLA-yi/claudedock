# 51-03 QUAL-03 — `verifyDNS` 遍历全部 nameserver 行

## Goal

加固 `internal/network/verify.go::verifyDNS`，使其在 `result.ActualDNS` 字段
上不止暴露**第一行** nameserver，而是把 `/etc/resolv.conf` 所有 nameserver 行
汇总报告，确保 fallback nameserver（例如多 nameserver 配置）也能被审计断言识别。

**约束**：当前实现已经做整 buffer `rawContent != resolvConfContent` 严格比对
（Phase 45 WR-07，参考 verify.go:199-202），这一行为不变；本 plan 只扩展
`ActualDNS` 的报告维度，便于上层日志 / metadata 看到全部 nameserver，而不是
silently 截断。

## Files

- `internal/network/verify.go`（改）—— `verifyDNS` 在保留现有严格 byte 相等
  断言的同时，提取**所有 nameserver 行**到 `result.ActualDNS`（`","` 分隔），
  并提取 `result.ActualDNSAll []string`（如需新字段）。
- `internal/network/verify_test.go`（改）—— 新增多 nameserver 解析的纯函数
  单测。

## Steps

1. 引入私有解析函数 `parseAllNameservers(rawContent string) []string`：
   - 按行扫描；trim 后 prefix `nameserver` → 收集第 2 段；
   - 返回顺序保持原文件顺序；空 → 返回 `nil`。
2. 改造 `verifyDNS`：
   - 拿到 `rawContent` 后调 `parseAllNameservers(rawContent)`；
   - `result.ActualDNS` 写入「所有 nameserver 用逗号分隔」（如 `"172.19.0.1"` /
     `"172.19.0.1,8.8.8.8"`）—— 既能在单 ns 场景与旧值兼容（同 `firstNS`），又
     能在多 ns 场景把 fallback 暴露；
   - 严格比对仍然是 `rawContent != resolvConfContent`，因此 verifyDNS 的
     PASS / FAIL 行为不变。
3. 单测：
   - `TestParseAllNameservers_Empty / SingleNS / MultipleNS / Comments / Garbage`
     5 个 case。
   - `TestVerifyDNS_MultipleNameservers_ReportedAll`：fake nsenter 返回
     `"nameserver 172.19.0.1\nnameserver 8.8.8.8\n"` → `DNSCorrect=false` +
     `ActualDNS="172.19.0.1,8.8.8.8"`。

## Acceptance

- `go build ./...` + `GOOS=linux go build ./...` PASS。
- `go test ./internal/network/... -count=1` 全绿。
- 既有 `TestFirstNetworkError_DNSLeak{,_NilProxy}` 单测继续通过（断言
  `actual_dns="8.8.8.8"` —— fake 仅返回单 nameserver 行，`parseAllNameservers`
  也只返回一个元素，逗号分隔同样得到 `"8.8.8.8"`）。

## Risk & Mitigation

| Risk | Mitigation |
|------|-----------|
| `result.ActualDNS` 字段语义微变（单 ns 不变，多 ns 改为逗号分隔）。 | 既有所有 e2e + 单测断言场景都只关心「单 ns」（`172.19.0.1` 或 fake 桩 `8.8.8.8`），逗号分隔在单元素时退化为同一字符串，零回归。 |
| 严格 byte 比对仍是 PASS / FAIL 主信号，本 plan 不改其逻辑。 | 单测专注 parseAllNameservers + ActualDNS 字段；不修改 DNSCorrect 判定路径。 |
| 影响 51-01 / 51-02？ | 不交叉。 |
| 影响 e2e 用例？ | DNSCorrect 行为契约不变。 |
