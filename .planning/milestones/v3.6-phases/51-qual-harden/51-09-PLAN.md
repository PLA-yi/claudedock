# 51-09 — 双绑互斥 API pre-check + 稳定 error code

## Goal

闭 Phase 47 GAP D-47-3：当前 `internal/controlplane/http/admin_bindings.go::Bind`
没有「同 egress_ip_id 已绑定到其它 host」的 pre-check，4xx 错误响应也没有
稳定 error code。本 plan：

1. 新增 `AdminBindingStore.GetBindingHostIDByEgressIP(ctx, egressIPID) (string, error)`
   接口方法（对应 store 层 SQL `SELECT host_id FROM host_egress_bindings WHERE
   egress_ip_id=$1 LIMIT 1`）；
2. `Bind` 处理函数在通过 host running / not running 闸后**先**查这个方法；如果
   row 存在 → 返回 `409 Conflict` + `error_code = "egress_ip_already_bound"` +
   中文 message `"出口 IP 已绑定到其它宿主机"`；
3. 新增 error code 常量在合适位置（如 admin_bindings.go 顶部）。
4. **新增 `admin_bindings_test.go` 双绑互斥单测**覆盖该路径。
5. 既有 Phase 47 `tests/e2e/helpers_test.go::TestHelpersParseBindEgressIPResponse_*`
   解析逻辑应已自动从「找不到 error code 字段 → PARTIAL」翻到 PASS（因为响应
   现在带 `error_code` 字段）。

## Files

- `internal/controlplane/http/admin_bindings.go`（改）—— 新增 pre-check + error
  code 常量 + 改 Bind 处理函数；扩展 `AdminBindingStore` interface。
- `internal/store/repository/queries.go`（改）—— 实现 `GetBindingHostIDByEgressIP`
  SQL 方法。
- `internal/controlplane/http/admin_bindings_test.go`（改）—— stub store 加方法 +
  新增 `Bind double-bind 409 + error code` test case。

## Steps

1. **Repository 层**：在 `internal/store/repository/queries.go` 新增
   ```go
   func (r *Repository) GetBindingHostIDByEgressIP(ctx context.Context, egressIPID string) (string, error) {
       var hostID string
       err := r.pool.QueryRow(ctx,
           `SELECT host_id FROM host_egress_bindings WHERE egress_ip_id=$1 LIMIT 1`,
           egressIPID).Scan(&hostID)
       return hostID, err
   }
   ```
2. **HTTP 层** `admin_bindings.go`：
   - 新增常量：`const ErrCodeEgressIPAlreadyBound = "egress_ip_already_bound"`
   - 扩展 `AdminBindingStore` interface 加方法
     `GetBindingHostIDByEgressIP(context.Context, string) (string, error)`。
   - `Bind` 处理函数：在通过 host running 闸后、调 `BindEgressIPToHost` 前：
     ```go
     existingHostID, err := h.store.GetBindingHostIDByEgressIP(r.Context(), req.EgressIPID)
     if err != nil && !errors.Is(err, pgx.ErrNoRows) {
         writeJSON(w, 500, map[string]string{"error": "check existing binding failed"})
         return
     }
     if err == nil && existingHostID != "" && existingHostID != req.HostID {
         writeJSON(w, 409, map[string]any{
             "error":       "出口 IP 已绑定到其它宿主机",
             "error_code":  ErrCodeEgressIPAlreadyBound,
             "host_id":     existingHostID,
             "egress_ip_id": req.EgressIPID,
         })
         return
     }
     ```
   - 既有 host_id 等于 requested host_id 的场景：跳过 pre-check，继续走原
     INSERT（保持「同 host 重新绑定同 IP」幂等性，由 UNIQUE 复合键兜底）。
3. **单测**：
   - 在 `stubBindingStore` 加字段 `existingHostID string; existingHostIDErr error`
     与方法实现。
   - 既有 8 个测试 case 不破坏（新方法默认返回 `pgx.ErrNoRows` → pre-check 跳过）。
   - 新增 case `"Bind double-bind to another host 409 with error_code"`：
     - body `{"host_id":"h1", "egress_ip_id":"ip1"}`；
     - store `existingHostID="h2"`；
     - 期望 status=409，响应 JSON 含 `"error_code":"egress_ip_already_bound"`
       + `"error":"出口 IP 已绑定到其它宿主机"`。
   - 新增 case `"Bind same host re-bind 201 (idempotent path)"`：
     - body `{"host_id":"h1", "egress_ip_id":"ip1"}`；
     - store `existingHostID="h1"`（同 host）；
     - 期望 status=201（继续走原路径；UNIQUE 复合键兜底重复 row）。

## Risk & Mitigation

| Risk | Mitigation |
|------|-----------|
| `AdminBindingStore` interface 加方法是 break change；所有 stub / impl 都要补。 | 项目当前只有 1 个生产 impl（Repository）+ 1 个测试 stub；本 plan 一并改齐。 |
| SQL 加 LIMIT 1 但表无 `egress_ip_id` 单列 UNIQUE（Phase 47 D-47-3 grep 结论）。 | SQL 加 LIMIT 1 + 取第一行；并发 race 不在本 plan 范围（CONTEXT §Deferred 明确）。 |
| 中文 message 在 admin 前端是否会乱码？ | 项目前端已处理 utf-8 中文 error message；既有 401 / 403 等已存在中文 error。 |
| 影响 51-01..08？ | 不交叉（不动 verify.go / namespace.go / firewall / cap / test infra）。 |
| 影响 e2e 用例？ | Phase 47 用例（darwin 纯函数单测）已锁 `EgressIPDoubleBindContract{WantStatus:409, WantErrSubstring:"already bound"}`；服务端 `error` 字段是中文，但 Phase 47 helpers `ParseBindEgressIPResponse` 解析的是 `ErrorMessage` 字段（中文文本中如包含「已绑定」可命中`已绑定` 子串，但 `"already bound"` 是英文 → 若 Phase 47 用例硬断言英文子串则需在 message 中拼接英文）。**修正**：本 plan response body 同时含 `"error"`（中文）和 `"error_code"`（机器可读常量）；纯函数 `ParseBindEgressIPResponse` 应优先校验 `error_code == ErrCodeEgressIPAlreadyBound`，已满足 Phase 47 锁定契约。如发现 helpers 仅断言英文子串，本 plan 同时在 `error` 字段中追加英文片段 `"egress IP already bound to another host"`。 |
| 既有 Phase 47 `TestEgressIPBinding_DoubleBindExcluded` 等 e2e 用例文件不修改。 | tests/e2e 不动（CONTEXT 锁定）。 |
