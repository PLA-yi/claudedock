# 51-08 QUAL-08 — `goleak.VerifyTestMain` 接入

## Goal

接入 `go.uber.org/goleak` 拦截 goroutine 泄漏。本 plan 是**本里程碑唯一允许引入
新依赖**（go.mod 加 `go.uber.org/goleak`）。

## Files

- `go.mod` / `go.sum`（改）—— 新增 `go.uber.org/goleak`。
- `internal/controlplane/app/testmain_test.go`（新建）—— `TestMain` +
  `goleak.VerifyTestMain` + ignore list。
- `cmd/claudedock/testmain_test.go`（新建）—— 同上。
- `internal/network/testmain_test.go`（新建）—— 同上（network 包有 sing-box 调用，
  最需要 goleak）。

**理由**：项目主二进制 `cmd/control-plane` 与 `cmd/host-agent`（embedded
in-process）共享 `internal/controlplane/app`，没有独立 `cmd/host-agent` 目录；
goleak 接入 3 个最关键的内部包足以覆盖 sing-box / pgx 主路径。

## Steps

1. `go get go.uber.org/goleak@latest`（仅本 plan 允许新增依赖）。
2. 跑一遍 `go test ./internal/controlplane/app/... ./cmd/claudedock/...
   ./internal/network/... -count=1` 获取目前 goroutine leak top function 列表。
3. 各 `testmain_test.go`：
   ```go
   package <pkg>

   import (
       "testing"
       "go.uber.org/goleak"
   )

   func TestMain(m *testing.M) {
       goleak.VerifyTestMain(m,
           goleak.IgnoreTopFunction("github.com/jackc/pgx/v5/pgxpool.(*Pool).backgroundHealthCheck"),
           goleak.IgnoreTopFunction("github.com/sagernet/sing-box/..."), // 视实际包路径调整
           // 其它已知泄漏 top function（首跑后补充）
       )
   }
   ```
4. 若本 plan 跑出未知泄漏：
   - 优先把 top function 加入 IgnoreTopFunction（CONTEXT 明确允许）；
   - 不修真泄漏（属后续）。
5. 不在 `cmd/control-plane` / `internal/runtime/tasks` 等无 test 文件的包接入。

## Acceptance

- `go.mod` 新增 `go.uber.org/goleak`（且 only this one）；`go.sum` 同步。
- 3 个新 testmain_test.go 文件就绪。
- `go test ./... -count=1` 全绿（goleak 不误报）。

## Risk & Mitigation

| Risk | Mitigation |
|------|-----------|
| goleak 在并发 `t.Run` / 子测试 cleanup 顺序下误报。 | `VerifyTestMain` 模式只在测试结束统一检查，比 `VerifyNone` 宽松；常见误报靠 IgnoreTopFunction 兜。 |
| 新依赖 `go.uber.org/goleak` 是项目第一个 uber 依赖，go.sum 增长。 | CONTEXT 已明确允许；这是本里程碑唯一新依赖。 |
| ignore list 太宽 → 失去检测意义。 | 仅放已知第三方常驻 goroutine（pgx / sing-box）；项目自有 goroutine 不 ignore。 |
| 影响 51-07？ | -race / -shuffle 与 goleak 正交，可共存。如果 goleak 找出泄漏，shuffle 改变运行顺序可能让某些 ignore 不稳定 → 实测调整。 |
| 影响其它 plan？ | 仅加测试 infra，不动生产代码；与 01..06 / 09 无交叉。 |
| 影响 e2e 用例？ | tests/e2e 不加 testmain_test.go；无回归。 |
