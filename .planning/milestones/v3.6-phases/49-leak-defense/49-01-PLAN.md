# Phase 49 Plan 01: LEAK-01 DNS 明文 UDP/53 旁路检测

## Goal

证实 worker 容器内任何「绕过 sing-box，直接对公共 resolver `8.8.8.8` 走 UDP/53 明文 DNS」的尝试都无法在宿主机 eth0 上看到一个真实包；同时 `dig` 自身必须以 timeout / SERVFAIL / refused 退出，不允许被 tun 静默接管后返回正常 A 记录（区分 Phase 46 MVS-03 的 OR 语义 —— 这里要求**纯防泄漏**断言）。

落地为一个用例 `TestLeak_01_DNSPlainUDP_BlockedByHostEth0`：

- 容器内 `dig +short +time=3 +tries=1 @8.8.8.8 example.com` 必须**非 0 退出**或返回空（视作 Denied）。
- host eth0 抓包 BPF：`udp port 53 and dst host 8.8.8.8 and src host <workerIP>` 必须 0 包。

**不在范围**：

- `kdig +tls @1.1.1.1 example.com` → DoT 旁路（Plan 02）。
- 系统 stub resolver `127.0.0.53` 路径（覆盖在 Phase 47 / Phase 48 已锁定）。

## Steps

1. **shared 层落地**（本 plan 与 02..08 公用，统一在 `feat(49-shared)` 一笔合入）：
   - `LeakProbeResult{Blocked bool, Reason string, RawStdout, RawStderr string, ExitCode int, Duration time.Duration}`。
   - `LeakVerdict` 三值枚举：`LeakVerdictPass` / `LeakVerdictFail` / `LeakVerdictInconclusive`。
   - `ClassifyLeakProbe(result *LeakProbeResult, expectBlocked bool) LeakVerdict`：`Blocked == expectBlocked` → Pass；`Blocked == false && expectBlocked == true` → Fail；`result == nil` 或 ExitCode==-1 且 Reason 空 → Inconclusive。
   - 单测覆盖 5 分支（Pass × 2 / Fail × 1 / Inconclusive × 2）。

2. **`*GoldenPath.DigPlainDNS`**（带 tag）：
   - `docker exec <worker> dig +short +time=3 +tries=1 @<server> <name>`。
   - 解析 exit code + stderr：
     - exit 0 且 stdout 命中 IPv4 → `Blocked=false, Reason="resolved"`（这就是 LEAK，oracle 还要再确认抓包）。
     - exit 0 stdout 空 / exit 9 (NXDOMAIN/SERVFAIL) / exit 10 (timeout) / stderr 含 `connection timed out` → `Blocked=true, Reason="dig_timeout_or_servfail"`。
     - 容器句柄未填充 → 返 err `errWorkerContainerHandleUnavailable`。

3. **`leak/suite_test.go`**（共享 fixture）：
   - `LeakSuite` 嵌入 `harness.BaseSuite`；`SetupSuite` 调 `StartGoldenPath` 一次；后续 8 个 leak 用例共享一个 GoldenPath 实例（节省 8× 冷启动）。
   - `TearDownSuite` 调 GoldenPath 注册的 cleanup（已通过 t.Cleanup 自动）。
   - 注入 `ArtifactDumper`，让失败用例自动落 dump。

4. **主用例 `TestLeak_01_DNSPlainUDP_BlockedByHostEth0`**：
   - 起 host eth0 tcpdump goroutine，BPF：`udp port 53 and dst host 8.8.8.8 and src host <workerIP>`，count=5，window=5s。
   - 容器内 `g.DigPlainDNS(ctx, "8.8.8.8", "example.com")`。
   - 等 tcpdump 退出 → `tdRes.packets`。
   - 裁决：
     - `tdRes.packets > 0` → t.Fatalf（host eth0 抓到明文 53 包，实锤 leak）。
     - `digResult.Blocked == false && digResult.Reason == "resolved"` → t.Fatalf（dig 拿到了 A 记录，要么走了 tun 接管要么真泄漏；这里要求**纯阻断**语义，不接受 Tunneled）。
     - 否则 → Pass。
   - 失败时 t.Logf 打印 dig stderr 末尾 + workerIP + bpf。

5. **darwin 兜底**：用例文件带 `//go:build e2e && linux`；纯函数单测无 tag。

## Risk & Mitigation

| 风险 | 缓解 |
|------|------|
| worker 容器内没有 `dig`（managed-user 镜像 grep 未发现 `dnsutils`） | suite SetupSuite 检测：`docker exec <worker> which dig` 失败 → `apt-get update && apt-get install -y dnsutils`（在 testcontainer init 阶段一次性补齐），或退化用 `getent hosts example.com` 不指定 server，但失去 LEAK-01 语义 → 优先 apt 补齐。 |
| sing-box 把容器 53 流量重写到 tun 后真返回 A 记录（OR 语义合法） | 本 LEAK 严格不接受 Tunneled，必须验证 tcpdump 0 包；只要 host eth0 不漏，即便 dig 返回了 A 记录也只能算"被 tun 接管"——但本 plan 的契约要求 dig 自身也失败，否则视为防泄漏断言被绕过（PLAN 显式区分 Phase 46）。 |
| host eth0 在 macOS LinuxKit VM / 非特权 runner 抓不到包 | 复用 Phase 48 `TcpdumpOnHostEth0` 的 sidecar 路径 + `E2E_ALLOW_HOST_TCPDUMP` env；不可用 → t.Skip + VERIFICATION 标 deferred-to-CI。 |
