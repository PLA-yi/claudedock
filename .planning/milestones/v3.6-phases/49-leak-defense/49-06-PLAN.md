# Phase 49 Plan 06: LEAK-06 raw socket 拒绝

## Goal

证实 worker 容器内任何尝试创建 `SOCK_RAW` socket 的进程都必须以 `PermissionError: [Errno 1] Operation not permitted` 失败 —— 即 worker capability 集合里**不**含 `CAP_NET_RAW`。

`TestLeak_06_RawSocket_PermissionDenied`：

- 容器内 `python3 -c 'import socket; socket.socket(socket.AF_INET, socket.SOCK_RAW, socket.IPPROTO_ICMP)'` 必须**非 0 退出 + stderr 含 `Operation not permitted` 或 `PermissionError`**。
- worker 缺 python3 → 回退 `bash -c 'exec 3<>/dev/raw/icmp 2>&1'`（fail），或 `unshare -r ping -c1 8.8.8.8` 验证（次选）。

**预期**：当前 worker.go grep 未显式 `--cap-drop NET_RAW`，docker 默认 capability 集合**包含** `cap_net_raw`，因此本用例**预期 fail** → 这是 Phase 51 QUAL-06 必须修的 backend GAP。VERIFICATION 标 gap，本 plan 不阻塞。

## Steps

1. **`*GoldenPath.TryRawSocket`**：
   - 优先 `docker exec <worker> python3 -c 'import socket; s = socket.socket(socket.AF_INET, socket.SOCK_RAW, socket.IPPROTO_ICMP); print("RAW_OK")'`。
   - python3 缺失 → `bash -c 'python -c "..." 2>&1 || (exec 3<>/dev/raw/icmp 2>&1)'` 兜底（旧 ping 风格）。
   - 解析：
     - exit 0 + stdout 含 `RAW_OK` → `Blocked=false, Reason="raw_socket_allowed"`（实锤 leak）。
     - exit 1 + stderr 含 `PermissionError` / `Operation not permitted` → `Blocked=true, Reason="raw_socket_denied"`。
     - 其它 → Inconclusive。

2. **主用例 `TestLeak_06_RawSocket_PermissionDenied`**：
   - `g.TryRawSocket(ctx)` → `result.Blocked == true` 才 Pass。
   - `result.Blocked == false` → t.Errorf 而非 t.Fatalf（让其它 leak 用例继续跑），并把 reason / stderr 末尾打到 t.Logf；同时把这条**预期失败**记录到 SUMMARY 的「backend GAP」段。

3. **suite SetupSuite 兜底**：worker 缺 python3 → `apt-get install -y python3-minimal`（非阻塞，失败回 bash exec 路径）。

4. **darwin 兜底**：build tag。

## Acceptance

darwin：纯函数单测 + build / lint PASS。Linux：用例**预期 fail**（gap），VERIFICATION 标 `gaps_found`；只要 fail reason 是 `raw_socket_allowed`，就明确指向 worker.go capability 缺 drop。Phase 51 QUAL-06 修源码后预期转 Pass。

## Risk & Mitigation

| 风险 | 缓解 |
|------|------|
| worker 镜像无 python3（grep managed-user/Dockerfile 未见 python） | suite 兜底 apt 安装；安装失败也用 `exec 3<>/dev/raw/...` bash 路径，行为模式一致（permission denied = blocked）。 |
| docker 默认 capability 集合在不同 docker 版本/平台略有差异（NET_RAW 默认通常包含） | 用例不区分 backend 版本；只看实际行为，由抓包 / dump 提供证据。 |
| 与 LEAK-08（capability 审计）耦合 | LEAK-08 是 oracle 角度（看 /proc/1/status CapEff/CapBnd），LEAK-06 是行为角度（实际 SOCK_RAW 创建是否成功）；两层独立 oracle 互证。 |
