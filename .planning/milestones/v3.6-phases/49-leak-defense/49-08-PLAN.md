# Phase 49 Plan 08: LEAK-08 capability 审计

## Goal

证实 worker 容器进程 1 的 capability 集合中 `CapEff` 与 `CapBnd` **均不**含 `CAP_NET_RAW` / `CAP_NET_ADMIN` / `CAP_SYS_ADMIN`。

`TestLeak_08_WorkerCapabilities_Locked`：

- 容器内 `cat /proc/1/status | grep -E 'CapEff|CapBnd|CapInh|CapPrm'` 解析为 `Set[Capability]`。
- 断言：`CapEff ∩ {NET_RAW, NET_ADMIN, SYS_ADMIN} == ∅` 且同样 `CapBnd`。

**预期**：grep `internal/runtime/tasks/worker.go:217-218` 确认当前 worker create args 显式 `--cap-add NET_ADMIN --cap-add SYS_ADMIN`，且**未**显式 `--cap-drop NET_RAW`（docker 默认 caps 含 NET_RAW）→ 本用例**预期 fail**，gap 列入 Phase 51 QUAL-06。

## Files to touch

- `tests/e2e/helpers.go`（shared commit）：
  - `Capability` 字符串常量集（≥5 个常用：`CAP_NET_RAW` / `CAP_NET_ADMIN` / `CAP_SYS_ADMIN` / `CAP_SYS_PTRACE` / `CAP_DAC_READ_SEARCH`）。
  - `ParseProcCapabilities(raw string) (capEff, capBnd, capInh, capPrm map[string]bool, err error)` —— 解析 `/proc/<pid>/status` 中 `CapXxx: <hex>` 行，把 64 位掩码展开为已知 capability 名集合。
  - `KnownCapabilityBits` 锁定表（位 → 名字），覆盖 ≥10 个 cap（够覆盖 NET_RAW/NET_ADMIN/SYS_ADMIN/CHOWN/DAC_READ_SEARCH/FOWNER/KILL/SETUID/SETGID/SYS_PTRACE）。
- `tests/e2e/helpers_linux.go`（shared commit）：
  - `*GoldenPath.GetProcCapabilities(ctx, pid int) (string, error)` —— `docker exec <worker> cat /proc/<pid>/status`，返回 raw。
- `tests/e2e/leak/leak_08_capability_test.go`（带 tag，新文件）。
- `tests/e2e/testdata/leak/proc_status_clean.txt` / `proc_status_dirty.txt` / `proc_status_partial.txt`：fixture。

## Steps

1. **`KnownCapabilityBits` + `ParseProcCapabilities`**（纯函数）：
   - 锁定常量表（bit index → cap name）：参照 `<linux/capability.h>`，覆盖 ≥10 个 cap。
   - 行扫 `/proc/<pid>/status`：
     - `CapInh:	0000000000000000` 等 4 行；hex 部分 16 chars。
     - 把 hex → uint64，按位 AND `KnownCapabilityBits`，命中位的 cap name 加入 set。
   - 解析失败（行数 < 4 / hex 长度异常）→ err。
   - 单测覆盖 4 fixture（clean / dirty / partial / 损坏 hex）。

2. **`*GoldenPath.GetProcCapabilities`**：
   - `docker exec <worker> cat /proc/<pid>/status`。
   - 返回完整 stdout；exit !=0 → err。

3. **主用例 `TestLeak_08_WorkerCapabilities_Locked`**：
   - `raw, err := g.GetProcCapabilities(ctx, 1)`。
   - `eff, bnd, _, _, err := ParseProcCapabilities(raw)`。
   - 断言 `eff["CAP_NET_RAW"] == false && eff["CAP_NET_ADMIN"] == false && eff["CAP_SYS_ADMIN"] == false`，同样 `bnd`。
   - 任一命中 → t.Errorf（不 Fatalf），t.Logf 把 hex + 命中的 cap 列出，SUMMARY 标 backend GAP。

4. **darwin 兜底**：build tag；纯函数单测 ≥ 4 个 fixture × 3 断言 ≈ 6+ 个。

## Acceptance

darwin：纯函数单测 + build / lint PASS。Linux：用例**预期 fail**（worker 当前带 NET_ADMIN/SYS_ADMIN/默认 NET_RAW）；fail 即明确指向 Phase 51 QUAL-06 修源码。

## Risk & Mitigation

| 风险 | 缓解 |
|------|------|
| `/proc/<pid>/status` 字段顺序在不同内核略有差异 | 解析器按行扫前缀字面量 `CapEff:` 等，不假设顺序；缺任一 → err（让用例显式失败）。 |
| docker 默认 capability 集合在不同 docker 版本变动 | 直接读 `/proc/1/status` 是 ground truth，与 docker 默认无关；用例 oracle 是内核视角，最稳。 |
| 用例与 LEAK-06 重叠（LEAK-06 行为视角 / LEAK-08 oracle 视角） | 两条 LEAK 各自独立失败 / 通过；都依赖 worker.go 显式 `--cap-drop` 修复，预期 Phase 51 QUAL-06 一并 close。 |
