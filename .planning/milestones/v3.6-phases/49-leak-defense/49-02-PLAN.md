# Phase 49 Plan 02: LEAK-02 DoT (853) 旁路检测

## Goal

证实 worker 容器内任何「绕过 sing-box，直接对公共 DoT resolver `1.1.1.1:853` 建立 TLS 连接」的尝试都无法在宿主机 eth0 上看到一个真实 TCP 握手包。

落地为 `TestLeak_02_DoT853_BlockedByHostEth0`：

- 容器内 `kdig +tls +time=3 @1.1.1.1 example.com`（缺 kdig 时回退 `openssl s_client -connect 1.1.1.1:853 -brief </dev/null`）必须**非 0 退出 / TLS 握手失败 / connect timeout**。
- host eth0 抓包 BPF：`tcp port 853 and dst host 1.1.1.1 and src host <workerIP>` 必须 0 包。

**不在范围**：UDP/53 明文（Plan 01）；DoH (443) 旁路（v2 范围）。

## Files to touch

- `tests/e2e/helpers_linux.go`（shared commit）：
  - `*GoldenPath.DigDoT(ctx, server, name string) (*LeakProbeResult, error)` —— 优先 `kdig +tls`；缺失时 fallback `openssl s_client`。
- `tests/e2e/leak/leak_02_dot_853_test.go`（带 tag，新文件）：主用例。
- `tests/e2e/testdata/leak/kdig_tls_timeout.txt` / `openssl_s_client_refused.txt`：fixture。

不动：`internal/network/` 任何源码。

## Steps

1. **`*GoldenPath.DigDoT`**：
   - `docker exec <worker> bash -c 'command -v kdig >/dev/null && kdig +tls +time=3 @<server> <name> || (timeout 5 openssl s_client -connect <server>:853 -brief </dev/null)'`。
   - 解析：
     - exit 0 且 stdout 含 IPv4 / `Verify return code: 0 (ok)` → `Blocked=false, Reason="dot_handshake_ok"`（实锤 leak）。
     - 任何其它 exit / stderr 含 `Connection refused` / `unable to load` / `timed out` / `verify error` → `Blocked=true`。

2. **suite SetupSuite 兜底**：检测 `which kdig || which openssl`；都缺 → `apt-get install -y knot-dnsutils openssl`（仅 worker 容器一次性）。

3. **主用例 `TestLeak_02_DoT853_BlockedByHostEth0`**：
   - 起 host eth0 tcpdump goroutine：BPF `tcp port 853 and dst host 1.1.1.1 and src host <workerIP>`，count=5，window=5s。
   - `g.DigDoT(ctx, "1.1.1.1", "example.com")`。
   - 等 tcpdump 退出。
   - 裁决：
     - `tdRes.packets > 0` → t.Fatalf（实锤 853 直连泄漏）。
     - `dotResult.Blocked == false` → t.Fatalf。
     - 否则 → Pass。

4. **darwin 兜底**：用例文件带 build tag。

## Acceptance

同 Plan 01。新增 e2e 用例 1 个（不新增纯函数单测，复用 Plan 01 的 `ClassifyLeakProbe`）。

## Risk & Mitigation

| 风险 | 缓解 |
|------|------|
| worker 镜像无 kdig | `openssl s_client` 兜底，失败模式同样表达为 connect refused / handshake timeout；实在两个都缺就 apt-get install。 |
| `1.1.1.1:853` 被 sing-box 路由接管返回正常 TLS 握手（tun 视角） | 本 LEAK 严格要求**直连**失败 + host eth0 0 包；即便 tun 内 TLS 握手成功，也不会出现在 eth0 BPF（因为 tun 是 sb-tun0 而非 eth0）。 |
| openssl 在某些镜像上 verify 失败但 connect 成功 | 把 stdout 中 `connect:` 后的字面量也喂 ClassifyLeakProbe；只要 connect 成功且不在 stderr 出现 refused/timeout 就视为可疑 leak。 |
