# Quick: 用户中心化凭据重构 + 三项捆绑修复 (260502-ni0)

## 背景与目标
- 把 `entry_password` 与 `ssh_keypair` 的所有权从 host 上提到 user 上，且强约束「一用户一活跃 host」。
- 同步处理三个独立小修复：容器自启动、删除 claude-real wrapper、IP 测试镜像复用。

## 用户决策
- 存量「一用户多 host」：不存在 → 直接加 partial unique index。
- 存量用户没 SSH key：靠用户详情页「重新生成 SSH 凭据」按钮，不在迁移里补。
- `hosts.entry_password` 直接 DROP，不留兼容。

<read_first>

- internal/store/migrations/0008_host_ssh_identity.sql
</read_first>

<action>
新建 `internal/store/migrations/0018_user_centric_credentials.sql`：

```sql
-- 0018: 用户中心化凭据 — 把 entry_password 提升到 users，强约束一用户一活跃 host。
BEGIN;

-- 反向回填：原 host.entry_password → user.entry_password（仅当 user 侧为空且 host 侧非空）
UPDATE users u
SET entry_password = h.entry_password
FROM hosts h
WHERE u.id = h.user_id
  AND COALESCE(u.entry_password, '') = ''
  AND COALESCE(h.entry_password, '') <> '';

-- 一用户一活跃 host 硬约束（排除 deleted/archived）
CREATE UNIQUE INDEX idx_hosts_user_active
  ON hosts (user_id)
  WHERE status NOT IN ('deleted', 'archived');

-- 删除 hosts.entry_password 列
ALTER TABLE hosts DROP COLUMN entry_password;

COMMIT;
```
</action>
