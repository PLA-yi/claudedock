# Quick Task 总结：claudedock ssh doctor

## 背景

前置 quick task `260416-wvu` 把 `injectSSHKeys` 改成幂等之后，控制面不会再覆盖用户手放的 key 了。但排查同一天另一场故障时发现，用户遇到的根因其实**不是控制面写覆盖**，而是：

- `/workspace/.ssh/vision` 文件 owner 是 `root:root`（容器内当前用户读不到）。
- 同一份 key 末尾缺少 `\n`，OpenSSH 走 libcrypto 解析失败，整条链路报 `error in libcrypto`。

这两个问题都落在"容器内 SSH 目录的健康状况"这层，既不是服务端 bug，也不是加密传输问题。本 quick task 给 `claudedock` CLI 加一个可复用的自检命令，下次再遇到就一眼可见、一键可修。

## 关键设计决策

### 1. 一次 SSH session 扫完所有文件
扫描阶段通过 `sess.Run(scanScript)` 单次往返，脚本用 `find . -print0 \| sort -z \| while read -d ''` 遍历 `/workspace/.ssh/` 下所有常规文件，按行输出 `FILE|<rel>|<owner>|<mode>|<firstline>|<lastbyte_hex>` 的定长结构。首行用 `tr -d '|'` 预处理去掉 `|`，保证解析器安全。对比"每个文件一次 `docker exec` / SSH session"的拍脑袋方案，连接开销从 O(N) 降到 O(1)。

### 2. 解析抽成纯函数 `parseScanOutput(raw string)`
`RunSSHDoctor` 内部 = `sshConnect` + `runSSHSession` + `parseScanOutput` + 可选 `applyFixes`。解析拆出来后，`ssh_doctor_test.go/parse_scan_output` 能直接喂构造字符串做断言，不需要 mock SSH。`parseScanOutput` 本身无副作用、不依赖网络。

### 3. 修复用 `shellescape.Quote` 而不是字符串拼接
用户名 / 路径都通过 `al.essio.dev/pkg/shellescape.Quote`（项目已在 `ssh.go` 里用过）做 shell quoting，避免含空格的文件名或特殊字符导致命令被截断。这是一个已经在仓库中固化的模式，保持一致。

### 4. sudo 降级明确、不悄悄失败
`applyFixes` 里区分两种修复路径：
- `chown` 必然需要 `sudo`（因为 owner 被改了就说明不是当前用户拥有）→ 先查 `r.SudoOK`，不可用就只写 `FixFailed` 提示手工命令，不挣扎不静默。
- `chmod` / 追加换行先走普通用户路径，失败且 `SudoOK` 可用才降级 `sudo`；再失败就写 `FixFailed`。

这样用户看报告永远能分清"修没修"与"为什么没修"。

### 5. `Print` 风格对齐 `EnvCheckResult.Print`
复用 `envcheck.go` 里的 `section` / `row` 辅助（同包内可见）。标题框、分组、✓/✗/⚠ 符号的视觉语言完全一致 —— 新命令对用户是"已有习惯的延伸"，不是"又一种新 UI"。
