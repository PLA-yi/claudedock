<objective>
修复 Docker 重启后出口 IP 泄漏问题。

Purpose: 当前 user 容器和 gateway 容器都使用 `--restart unless-stopped`，Docker daemon 重启后它们会自动重启，但 sing-box gateway 的网络栈（自定义 bridge、iptables、tun 设备）不会被重建，导致容器出网流量绕开隧道、直接泄漏。本 plan 将 restart 策略改为 `no`，并在控制面启动时通过对账机制自动恢复 DB 中标记为 running 的主机。

Output: 修改后的 worker.go、container_proxy_provider.go、reconciler.go 及对应测试。
</objective>
