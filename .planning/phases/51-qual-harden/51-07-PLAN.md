# 51-07 QUAL-07 — `go test ./... -race -shuffle=on -count=1` 默认

## Goal

把「Go 主测试命令」从 `go test ./... -count=1` 升级到 `go test ./... -race
-shuffle=on -count=1`，由 race detector 与 shuffle 共同把潜伏 race / 测试间
状态污染暴露出来。`tests/e2e/` 不带 `-race`（性能开销 + e2e 跑 docker，不适合
race detector）。

## Files

- `Makefile`（改）—— `test-go` target 加 `-race -shuffle=on`；同时保持 `tests/e2e`
  排除（既有 `go test ./...` 默认含 tests/e2e，要么用 `-skip` 要么 `go list`
  排除）。
- `.github/workflows/ci.yml`（改）—— `Run Go tests` 步骤加 `-race -shuffle=on`。

## Steps

1. **Makefile** `test-go` target：
   ```
   test-go:
       go test $$(go list ./... | grep -v /tests/e2e) -race -shuffle=on -count=1
   ```
   保留 `tests/e2e` 独立 target（如已存在，不动；如不存在，本 plan 不新增）。
2. **ci.yml** `go-test` job：
   ```yaml
   - name: Run Go tests
     run: go test $(go list ./... | grep -v /tests/e2e) -race -shuffle=on -count=1
   ```
3. **ci-gate** target 也同步加 `-race -shuffle=on`（如不破坏 `-short` 语义）。

## Acceptance

- `make test-go` 本地 darwin 跑全绿（race detector + shuffle 不引入新 fail）。
- `go test ./... -race -shuffle=on -count=1` 跑（去 tests/e2e）darwin 跑全绿。
- ci.yml `Run Go tests` step 变更 + 在 PR 上 CI 跑通（CI 无关阻塞）。

## Risk & Mitigation

| Risk | Mitigation |
|------|-----------|
| race detector 暴露既存 race（既有未 catch 过）。 | 若发现真 race，本 plan **不修真 race**（属后续）；先在 SUMMARY 标记，必要时回滚本 plan 等修完 race。 |
| shuffle 暴露依赖测试顺序的 bug。 | 同上：若发现，记录在 SUMMARY，本 plan 视情况回滚部分 shuffle 配置。 |
| -race 二进制要求 cgo enabled。 | Go 1.25 默认 cgo enabled；CI ubuntu-latest 默认有 gcc，可正常跑 -race。 |
| 影响 51-08？ | 51-08 加 goleak.VerifyTestMain，与 -race / -shuffle 正交，可共存。 |
| 影响其它 plan？ | 不交叉（只改 Makefile / CI 配置）。 |
| 影响 e2e 用例？ | tests/e2e 显式排除，无回归。 |
