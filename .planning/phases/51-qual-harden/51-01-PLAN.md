# 51-01 QUAL-01 — `verifyEgressIP` 多源轮询

## Goal

把 `internal/network/verify.go::verifyEgressIP` 从「单源 `https://ip.me`」改造成
「3 源固定轮询 + 多数派判定」，与 Phase 46 `Vote` 多数派语义对齐。语义不变（任一
PASS 路径下 `EgressIPMatch=true`），但行为更稳健 —— 单源 5xx/429/network blip
不再导致整条链路误判 leak。

接口契约保护：`verifyEgressIP` 旧签名保留；新增的 `verifyEgressIPMulti`
（包内私有）由 `VerifyNetworkIntegrity` 调用，对外行为透明升级。

## Files

- `internal/network/verify.go`（改）—— 新增 source 列表常量 + `verifyEgressIPMulti`
  内部实现 + 投票函数；保留 `verifyEgressIP` 旧签名作为 backward-compat shim。
- `internal/network/verify_test.go`（改）—— 在不破坏既有断言的前提下，新增多数派
  + tie / 全 timeout / 部分失败的纯函数单测。

## Steps

1. 在 `verify.go` 顶部新增包级常量切片 `egressIPSources = []string{"https://ip.me",
   "https://ifconfig.io", "https://ipinfo.io/ip"}`（与 Phase 46 一致）。
2. 抽出包内私有 `voteEgressIP(results []string) (winner string, ok bool)`：
   - 输入：去除 trim 空白后 ≥2 个一致 = 多数派；
   - 全为空 / tie → `ok=false`。
   - 这是 Phase 46 `Vote` 的 production-side 等价物（**不能直接 import tests/e2e
     包**）；语义与 `tests/e2e/helpers.go::Vote` 完全对齐。
3. 新增 `verifyEgressIPMulti(ctx, prefix, expectedIP, sources, result)`：
   - 用 `errgroup` / WaitGroup 并发 curl 3 个 source；每个 source 单独 8s timeout；
   - 收集结果 → `voteEgressIP` → 写入 `result.ActualEgressIP / EgressIPMatch`。
4. 把 `VerifyNetworkIntegrity` 中 `verifyEgressIP(ctx, prefix, …, &result)` 调用换成
   `verifyEgressIPMulti(ctx, prefix, expected.ExpectedIP, egressIPSources, &result)`。
   旧 `verifyEgressIP` 函数仍保留（不删；后续 phase 可以删，避免本 phase 触发外部
   调用方破坏）。
5. 单测：
   - `TestVoteEgressIP_*`：覆盖 3-0 / 2-1 / 1-1-1 tie / 全 timeout / 一致非
     expected 等 5 个 case。
   - `TestVerifyEgressIPMulti_*`：通过 fake `nsenterRunner` 桩 3 个 source 的
     返回值，覆盖 majority match / majority mismatch / all timeout 三个分支。

## Acceptance

- `go build ./...` 与 `GOOS=linux go build ./...` 通过。
- `go test ./internal/network/... -count=1` 全绿（含新增 5+3 个单测）。
- `verifyEgressIP`（旧签名）仍存在且行为不变（不破坏外部假设）。
- 既有 `TestVerifyNetworkIntegrity_*` 单测不需要修改即可通过（fake nsenter 桩
  机制不变）。

## Risk & Mitigation

| Risk | Mitigation |
|------|-----------|
| 3 个 source 并发 + 各自 8s timeout 导致整体最坏延迟从 15s → 8s（取最长一个）。 | 用 errgroup + ctx 父超时控制；最坏 ≤ 8s + 父 ctx 上限。 |
| Phase 46 `Vote` 在 tests/e2e 包不可在 production import。 | 在 verify.go 内私有 `voteEgressIP` 复刻同样语义；CONTEXT §「直接复用」与代码路径冲突，已在本 plan 注明偏差。 |
| 现有 fake nsenterRunner 桩只能返回一次值 → 多 source 并发会把它桩多次。 | 单测改造方向：fake 闭包根据 `args` 含 `ip.me` / `ifconfig.io` / `ipinfo.io` 关键字返回不同值；既有 4 个 TestVerifyNetworkIntegrity_* 用例若使用空桩仍可通过（返回 err → 全 source 失败 → vote fail → 与旧路径 `err != nil → EgressIPMatch=false` 同行为）。 |
| 影响其它 plan（51-02 / 51-03 / 51-09）？ | 仅改 `verifyEgressIP` 调用点；不动 verifyDNS / verifyLeakBlocked / admin_bindings。无交叉。 |
| 影响 e2e 用例？ | 行为契约不变（`EgressIPMatch` 字段语义 + 错误码不变）；Phase 46 vote 已锁定的 e2e 用例无回归。 |
