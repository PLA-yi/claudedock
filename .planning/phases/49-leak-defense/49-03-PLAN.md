# Phase 49 Plan 03: LEAK-03 ICMP 阻断

## Goal

证实 worker 容器内 `ping 8.8.8.8` 必须失败（exit code 非 0），且 host eth0 抓不到 ICMP echo-request 直连包。

`TestLeak_03_ICMP_BlockedByHostEth0`：

- 容器内 `ping -c 1 -W 3 8.8.8.8` 必须**非 0 退出**。
- host eth0 BPF：`icmp and dst host 8.8.8.8 and src host <workerIP>` 必须 0 包。

**不在范围**：ICMPv6 / Echo-Reply 反向流量（worker 不是 ICMP server）。

## Files to touch

- `tests/e2e/helpers_linux.go`（shared commit）：
  - `*GoldenPath.PingICMP(ctx, target string) (*LeakProbeResult, error)`。
- `tests/e2e/leak/leak_03_icmp_test.go`（带 tag，新文件）。
- `tests/e2e/testdata/leak/ping_perm_denied.txt` / `ping_unreachable.txt` / `ping_ok.txt`：fixture。

## Steps

1. **`*GoldenPath.PingICMP`**：
   - `docker exec <worker> ping -c 1 -W 3 <target>`。
   - 解析：
     - exit 0 + stdout 含 `1 received` → `Blocked=false`（leak）。
     - exit 1 + stdout 含 `0 received` → `Blocked=true, Reason="ping_no_reply"`。
     - exit 2 + stderr 含 `Operation not permitted` / `socket: Permission denied` → `Blocked=true, Reason="raw_socket_denied"`（这同时也覆盖 LEAK-06，但 Plan 06 单独验证 raw socket 自身）。
     - exit 1 + stderr 含 `Network is unreachable` / `Destination Host Unreachable` → `Blocked=true, Reason="route_unreachable"`。

2. **主用例**：
   - 起 host eth0 tcpdump goroutine：BPF `icmp and dst host 8.8.8.8 and src host <workerIP>`。
   - `g.PingICMP(ctx, "8.8.8.8")`。
   - 等 tcpdump。
   - 裁决：
     - `tdRes.packets > 0` → t.Fatalf。
     - `pingResult.Blocked == false` → t.Fatalf。
     - 其余 → Pass，t.Logf reason。

3. **darwin 兜底**：build tag。

## Acceptance

## Risk & Mitigation

| 风险 | 缓解 |
|------|------|
| `ping` 二进制需要 setuid 或 cap_net_raw | iputils-ping 在 ubuntu 24.04 默认 setuid；worker 镜像 grep 未发现 iputils-ping → suite SetupSuite 兜底 `apt-get install -y iputils-ping`。 |
| `ping` 因 raw socket 不可用而 exit 2（与 LEAK-06 重叠） | reason 标 `raw_socket_denied`，仍计 Blocked=true；Plan 06 用 SOCK_RAW 直接打头，不依赖 ping。 |
| sing-box 不接管 ICMP（kernel 默认丢弃从 tun namespace 出去的 raw ICMP），导致即便 nft 没显式 drop 也看不到包 | 这是合规结果，oracle 看 host eth0 0 包就 Pass，与 nft 行为是否显式无关。 |
