# Phase 49 Plan 07: LEAK-07 link-local 显式 drop

## Goal

证实 host / worker netns 上的 nftables 规则集中**至少有一条** destination 为 `169.254.0.0/16`（或更精确的 IMDS 子集如 `169.254.169.254/32` / `169.254.170.2/32`）的显式 drop 规则，作为 IMDS / link-local 路径的强约束。

`TestLeak_07_NftLinkLocalDrop_RuleExists`：

- `nft list ruleset` 输出经 `ParseNftRules` 解析得到 `[]NftRule`。
- 至少一条 `Action == "drop"` 且 `Dst` 字面量含 `169.254` 前缀。

**预期**：grep `internal/network/firewall_helpers.go` 仅有 IPv6 chain default drop + UDP dport drop（mDNS/LLMNR/NetBIOS）+ 链末 `addLogDropRule` 兜底，**未发现** 任何针对 IPv4 destination `169.254.0.0/16` 的显式 drop 规则 → 本用例**预期 fail**，gap 列入 Phase 51 QUAL-06 / QUAL-07。

> 注：链末 `sbfw-drop` 兜底虽然能在白名单后丢掉 169.254，但**不是显式 dst-CIDR 规则**，不满足本契约。实际抓包仍会 0 包（行为正确），但本 plan 关心的是「规则形态」是否显式存在便于审计。

## Steps

1. **`ParseNftRules`**（纯函数）：
   - 扫 `nft list ruleset` 输出，按 `table inet|ip|ip6 <name> { ... chain <name> { ... } }` 嵌套结构提取每条规则。
   - 对每行：
     - `ip daddr 169.254.0.0/16 ... drop` / `ip daddr 169.254.169.254 ... drop` → `NftRule{Action:"drop", Dst:"169.254.0.0/16", Proto:""}`。
     - `tcp dport 853 ... drop` → `NftRule{Action:"drop", Proto:"tcp", Port:853}`。
     - `udp dport 5353 ... drop` → `NftRule{Action:"drop", Proto:"udp", Port:5353}`。
     - `counter ... drop` 链末兜底 → 不归入「显式 link-local」（Dst 字面量为空）。
   - 不强追求 100% nft 语法覆盖；目标是给 LEAK-07 + LEAK-05 的 oracle 提供「是否有显式 link-local drop」的判定材料。
   - 单测覆盖 4 fixture（含正向 / 反向 / counter / 空）。

2. **`ParseNftCounters`**（纯函数）：
   - 解析 `counter packets <N> bytes <M>` 字面量；key 取规则的 `comment "..."`，否则用 `<chain>:<index>` 复合。
   - 单测覆盖 3 fixture（正向 / 多 counter / 缺 counter）。

3. **`*GoldenPath.ListNftRulesOnHost`**：
   - `docker run --rm --network host --cap-add NET_ADMIN --cap-add SYS_ADMIN <netshoot> nft list ruleset`。
   - host-native 路径：`E2E_ALLOW_HOST_TCPDUMP=1` + euid==0 → 直接 `nft list ruleset`。
   - 不可用 → 返 err，调用方 t.Skip。

4. **主用例 `TestLeak_07_NftLinkLocalDrop_RuleExists`**：
   - `raw, err := g.ListNftRulesOnHost(ctx)` → err 走 Skip。
   - `rules := ParseNftRules(raw)`。
   - 至少一条 `r.Action == "drop" && strings.HasPrefix(r.Dst, "169.254")` → Pass。
   - 否则 → t.Errorf（不是 Fatalf，让其它 leak 用例继续跑），明确把 raw nft 输出末尾段 + parsed rules 打到 t.Logf；SUMMARY 标 backend GAP。

5. **darwin 兜底**：build tag；纯函数单测 ≥ 7 个（4 ParseNftRules + 3 ParseNftCounters）。

## Acceptance

darwin：纯函数单测 + build / lint PASS。Linux：用例**预期 fail**（gap）。

## Risk & Mitigation

| 风险 | 缓解 |
|------|------|
| 现行 nft 规则只在 chain 默认 drop 而无显式 dst-CIDR 规则（grep 已确认） | 这正是要暴露的 gap，VERIFICATION 标 Phase 51 QUAL-06 修；用例 t.Errorf 不阻塞其它用例。 |
| nft 输出格式跨内核版本略有差异（行尾空格 / 缩进） | 解析器只关键 token 子串匹配，不依赖严格列对齐；fixture 用真机 ubuntu 24.04 / 22.04 各 1 份保证兼容。 |
| `nft list ruleset` 在非特权 sidecar 失败 | 需要 NET_ADMIN，用 `--cap-add NET_ADMIN` 启 sidecar；都失败 → t.Skip + Logf 打 docker stderr。 |
