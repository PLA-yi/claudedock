0它是什么

ClaudeDock 把你的 Claude Code 包装成「一个坐在美国家里、用普通电脑的开发者」，对抗 Anthropic 的风控封号；同时把每台容器绑定一个独立的住宅出口 IP，全流量走隧道出去。你 SSH 进容器就能写代码，本地项目目录会以完全相同的路径挂载进去，体验和本地无异。

三大核心

  • 身份伪装：CPU/MAC/machine-id 重写、Windows 主机名、容器痕迹擦除、太平洋时区、出口 TLS 指纹伪装成 Chrome、屏蔽遥测上报。
  • IP 严格隔离：每容器独立出口 IP，全流量走 sing-box tun 隧道，nftables 拒绝直连。支持 SOCKS5 / VMess / VLESS / Shadowsocks / Trojan / HTTP。
  • 代码同名映射：sshfs 把本地目录挂进容器，路径一致；断线自动重连、tmux 会话不丢。

架构一览

用户 ──curl──▶ 控制面 Control Plane (:8080) ──Docker──▶ ┌─ 用户容器 ─────────────┐ │ │ SSH + Claude Code + VNC │ SQLite（用户/主机/IP/审计） │ sshfs ← 本地同名路径映射 │ │ │ sing-box tun 隧道 │ 管理后台 (内嵌 SPA) │ ↓ 指定住宅出口 IP │ SSH 代理 (:2222) └────────────────────────┘
下文所有命令里，把 YOUR_HOST 换成控制面地址（本机就是 127.0.0.1，服务器就是它的 IP/域名），$TOKEN 是第 2 节第 1 步取到的管理员令牌。

1启动软件

软件本体 = 一个控制面容器。启动后浏览器访问管理后台即可。

A安装 Docker

# macOS / Windows：下载安装 Docker Desktop 并打开（等鲸鱼图标不再闪动）
# https://www.docker.com/products/docker-desktop/

# Linux：一行安装 Docker
curl -fsSL https://get.docker.com | sh

B生成配置（管理员密码 / 密钥）

cd ~/ClaudeProjects/Coding/claudedock-dev

# 交互式生成 .env（已有 .env 可跳过）。会打印 20 位管理员密码，请立刻记下
bash deploy/scripts/setup-env.sh
已有 .env 的话，账号是 admin，密码在文件里：grep ADMIN_PASSWORD .env

C启动控制面

Mac 最省事：双击项目根目录的 启动ClaudeDock.command（会自动本地构建 + 起服务 + 打开后台）。等价的手动命令如下（Mac / Linux 通用）：

# 1) 指定构建用的 compose 文件组合（避免拉取私有镜像）
export COMPOSE_FILE=docker-compose.yml:docker-compose.build.yaml:docker-compose.macbuild.yml

# 2) 本地从源码构建控制面镜像（首次较慢：容器内编译前端 + Go）
docker compose build control-plane

# 3) 启动
docker compose up -d control-plane

# 4) 验证健康
curl http://127.0.0.1:8080/healthz
# {"status":"ok","checks":{"database":"ok","agent":"ok"}}
别用 docker compose pull / 官方镜像——它是私有的，会报 error from registry: denied。一律走上面的本地构建。

D打开后台并登录

后台地址
http://localhost:8080（端口被占会自动换，见启动窗口）
用户名
admin
密码
.env 里的 ADMIN_PASSWORD

2管理员：开通一台可用主机

完整流程：登录 → 添加出口 IP → 创建用户 → 创建主机并绑定 → 分发接入命令。既可以在后台点按完成，也可以用下面的 API 命令（更精确、可脚本化）。

1获取管理员 Token

TOKEN=$(curl -s -X POST http://YOUR_HOST:8080/v1/auth/login \
  -H "Content-Type: application/json" \
  -d '{"username":"admin","password":"你的管理员密码"}' \
  | grep -o '"token":"[^"]*"' | cut -d'"' -f4)

echo "$TOKEN"   # 打印出来确认拿到了令牌

2添加出口 IP（住宅代理）

对应后台「出口 IP」页 → 新增。下面是 SOCKS5 示例（你买的 ISP 住宅代理就是这种）：

# SOCKS5（把 server/port/username/password 换成你的住宅代理）
curl -s -X POST http://YOUR_HOST:8080/v1/admin/egress-ips \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "label": "us-home-01",
    "ip_address": "149.119.153.132",
    "tunnel_type": "proxy",
    "provider": "manual",
    "proxy_config": {
      "type": "socks",
      "server": "149.119.153.132",
      "server_port": 443,
      "username": "gJGSQfYyRxXZ",
      "password": "jN4KpFpefR"
    }
  }'
# Shadowsocks 示例（换成对应字段即可）
curl -s -X POST http://YOUR_HOST:8080/v1/admin/egress-ips \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{
    "label": "jp-ss-01", "ip_address": "198.51.100.5",
    "tunnel_type": "proxy", "provider": "manual",
    "proxy_config": { "type": "shadowsocks", "server": "198.51.100.5",
      "server_port": 8388, "method": "aes-256-gcm", "password": "ss-password" }
  }'

测试这个出口能不能用（连通性 + 出口 IP 匹配 + DNS 泄漏三项）：

# 把 {ipID} 换成上一步返回的出口 IP 的 id
curl -s -X POST http://YOUR_HOST:8080/v1/admin/egress-ips/{ipID}/test \
  -H "Authorization: Bearer $TOKEN"

3创建用户

curl -s -X POST http://YOUR_HOST:8080/v1/admin/users \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{
    "username": "zhangsan",
    "password": "给用户的初始密码",
    "expires_at": "2026-12-31T23:59:59Z"
  }'

4创建主机并绑定出口 IP

# 创建主机（user_id 用上一步返回的用户 UUID）
curl -s -X POST http://YOUR_HOST:8080/v1/admin/hosts \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"user_id": "用户UUID"}'

# 绑定出口 IP（host_id、egress_ip_id 用前面返回的 UUID）
curl -s -X POST http://YOUR_HOST:8080/v1/admin/bindings \
  -H "Authorization: Bearer $TOKEN" -H "Content-Type: application/json" \
  -d '{"host_id": "主机UUID", "egress_ip_id": "出口IP的UUID"}'
主机至少绑定一个出口 IP 才能正常启动。后台「主机管理」页做这些是点按操作，效果一样。

5分发接入命令给用户

主机就绪后，在后台「主机详情」页复制接入命令，或直接把下面命令发给用户（替换 YOUR_HOST 和主机的 SHORT_ID）：

curl -sSf http://YOUR_HOST/entry/SHORT_ID | bash

3用户：接入并用 Claude Code

用户拿到管理员给的信息后，有两种接入方式。

A方式一：一条 curl 直接进（最快）

# 在自己电脑终端执行（YOUR_HOST 和 SHORT_ID 用管理员给的）
curl -sSf http://YOUR_HOST/entry/SHORT_ID | bash
# 按提示输入密码 → 等待容器就绪 → 自动 SSH 进入云主机
当前所在的本地目录会通过 sshfs 挂到容器内同名路径。断线后重新执行同一条命令即可恢复，容器不会因断开而停止。

B方式二：claudedock CLI（推荐，功能更全）

# 1) 安装（macOS / Linux，任选一种）
brew tap claudedock/tap && brew install claudedock
# 或一行脚本：
curl -fsSL https://raw.githubusercontent.com/claudedock/claudedock/main/scripts/install.sh | bash

# 2) 首次配置：填 网关地址 / Short ID / 密码（管理员给的三项）
claudedock init --gateway https://gw.example.com --short-id abc123 --password 你的密码

# 3) 进项目目录，直接用（当前目录自动挂载到容器同名路径）
cd ~/your-project
claudedock                      # 启动并连接
claudedock -p "帮我重构这个函数"   # 直接带任务

C会话 / 映射 / 自检

# 会话
claudedock --new-session        # 新建独立会话
claudedock --take-over          # 接管主会话，踢掉其他客户端
claudedock sessions             # 列出当前会话

# 映射模式
claudedock --mount-mode=auto        # 默认：HotSync 优先，失败降级 SSHFS
claudedock --mount-mode=sshfs-only  # 纯 SSHFS

# 排障自检
claudedock doctor               # 五维度全面自检
claudedock doctor mount --fix   # 检查挂载并自动修复
claudedock env check            # 检查远端时区 / 出口 IP / FUSE

D进容器后用 Claude

# 容器内已预装 Claude Code，直接：
claude
# 所有 Claude API 请求自动走你绑定的住宅出口 IP，无需额外配代理
容器内还预装了 Git / tmux / zsh / Node.js；KasmVNC + Chromium 远程桌面可在管理后台一键打开。

4进阶功能

1Bypass 防火墙（白名单直连）

默认全流量走隧道。若某些域名/网段要直连，在后台按域名、CIDR、端口加白名单，支持快照版本管理：预览应用回滚。

2远程桌面

容器内置 KasmVNC + Chromium，管理后台「主机详情」一键打开浏览器桌面。

3主机生命周期

后台可对主机：启动 / 停止 / 重建（保留或清空 /workspace）/ 删除。对应 API：

curl -s -X POST http://YOUR_HOST:8080/v1/hosts/{hostID}/start   -H "Authorization: Bearer $TOKEN"
curl -s -X POST http://YOUR_HOST:8080/v1/hosts/{hostID}/stop    -H "Authorization: Bearer $TOKEN"
curl -s -X POST http://YOUR_HOST:8080/v1/hosts/{hostID}/rebuild -H "Authorization: Bearer $TOKEN"
「重建」重置环境但保留 home 目录数据。

5日常运维命令

在项目根目录执行（务必先设 COMPOSE_FILE，否则会用错 compose）：

export COMPOSE_FILE=docker-compose.yml:docker-compose.build.yaml:docker-compose.macbuild.yml

docker compose ps                 # 查看状态
docker compose logs -f            # 实时日志
docker compose down               # 停止并移除

# 改了后台界面(React)或后端(Go)后，重新构建并更新：
docker compose build control-plane
docker compose up -d control-plane

·本地开发模式（不打包，热重载，需 Go + pnpm）

make setup    # 装前端依赖 + 生成 .env
make dev      # 起后端(内嵌) + 前端，前端 http://localhost:2568，API :8080
make dev-web  # 只起前端    
make dev-api  # 只起后端
make help     # 全部命令

6常见问题

Q启动报 error from registry: denied

在拉私有镜像。改用本地构建：

export COMPOSE_FILE=docker-compose.yml:docker-compose.build.yaml:docker-compose.macbuild.yml
docker compose build control-plane && docker compose up -d control-plane

QMac 上创建主机失败 / 容器网络异常

容器隔离依赖 Linux 宿主特性（pid:host、NET_ADMIN、tun）。Mac 上适合浏览后台、改界面；要真正开通用户容器，请部署到一台 Linux 服务器。

Q端口 8080 / 2222 被占用

启动器会自动改用相邻空闲端口（如 8082 / 2224），以启动窗口输出为准。手动指定：ADMIN_HOST_PORT=8090 SSH_PROXY_PORT=2230 docker compose up -d control-plane

Q忘记后台密码

grep ADMIN_PASSWORD .env

Q确认服务是否健康

curl http://127.0.0.1:8080/healthz
